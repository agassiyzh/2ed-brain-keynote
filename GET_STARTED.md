## 🛠️ Workshop 主持技巧

### Phase 1 最关键（定义 Schema）
- 给团队充足的讨论时间（不要赶时间）
- 鼓励他们大声说出：我们最常讨论什么？
- 记录在白板上，不要追求完美
- 提醒他们：第一版可以不完美，用 2 周后再迭代

### Phase 2 要演示
- 不要让他们自己 ingest，浪费时间
- 你自己示范一遍（5 分钟）
- 给他们看结果（10 张页面长什么样）
- 让他们感受到"哇，这可以运作"

### Phase 3 要鼓励
- 带他们尝试一个查询
- 如果成功了，他们会兴奋
- 如果不成功，解释原因，调整 Schema

## 💬 如果参会者提问...

**"这和 Notion/Confluence 有什么区别？"**
```
他们是静态存储。我们是动态维护。
LLM 在每次新信息来时自动更新、关联、修正。
你不需要手工保持一致性——LLM 会做。
```

**"我们小团队，真的需要这个吗？"**
```
特别适合。小团队知识损失的成本最高。
一个人离职，知识就永久丢失。
Wiki 保护你们的最大资产——人的智慧。
```

**"我们有很多技术债和过时决策，Wiki 会不会变垃圾？"**
```
很好的问题。这就是 Lint 检查的目的。
定期让 LLM 检查矛盾、过时信息。
旧决策标记为 Superseded，不删除（历史很重要）。
```

**"怎样防止 LLM 生成的内容有错？"**
```
人工审核。特别是关键决策、架构信息。
Schema 可以要求：重要信息需要 human approval 才能发布。
Git 版本历史让你可以随时回滚。
```
