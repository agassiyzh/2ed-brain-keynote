# slides.md 视觉美化设计

## 背景

现有 `slides.md` 已具备完整的 60 分钟内容主线，但视觉表达偏文字讲义：页面密度较高，缺少可记忆的视觉符号，关键概念如「知识流」「知识网络」「复利型记忆」主要依赖文字说明。

本次只美化 `slides.md`，不改动 README、Workshop 指南、Schema 模板或速查卡。

## 目标

将演示文稿升级为专业科技感的 Slidev 分享稿：

- 使用深色渐变、蓝紫绿色高光和知识图谱视觉，强化 AI 与团队记忆主题。
- 在关键页面加入必要图片、图示或内嵌 SVG，减少纯文字堆叠。
- 保留现有叙事结构和中文内容重点，避免大幅重写讲稿。
- 通过统一组件样式提升页面一致性、层次和演讲节奏。

## 视觉方向

采用「专业科技知识图谱风」：

- **基调**：深色背景、冷色渐变、霓虹节点、玻璃拟态卡片。
- **图形语言**：节点、连线、流程箭头、生命周期/阶段路径。
- **图片策略**：使用稳定在线图片作为封面和少数章节氛围图；核心解释用内嵌 SVG/HTML/CSS 图示，避免新增本地图片文件。
- **版式原则**：每页突出一个主观点，文字保留但通过卡片、对比栏、时间线和图谱结构分组。

## 改动范围

### 1. 全局 Slidev 样式

在 `slides.md` 中加入全局样式块，定义：

- 深色科技背景和正文色彩。
- 通用卡片、徽章、指标、步骤、图谱节点、流程箭头等 CSS 类。
- 统一标题层级、段落行距和列表间距。
- 适用于已有 Slidev/UnoCSS 写法的轻量类，避免引入新依赖。

### 2. 封面和收尾

重做封面页：

- 使用深色在线背景图或渐变背景。
- 加入「AI Knowledge Memory」类短标签。
- 加入知识网络装饰图，呼应第二大脑主题。

重做总结/最后一页：

- 用四阶段路径表达「有 Wiki → AI 维护 → 建立连接 → 团队共赢」。
- 用明确 CTA 引导观众进入 30 分钟 Workshop。

### 3. 核心解释页

重点优化以下类型页面：

- 问题页：把「知识去哪儿了」整理为问题卡片或隐性税图示。
- 传统 vs Wiki 对比页：改成左右对比卡片，弱化代码块式排版。
- LLM Wiki 运作页：加入「Sources → LLM → Wiki + Schema」流程图。
- 知识生命周期页：用阶段路径表达观察、摘要、规则、团队实践。
- 知识网络页：用节点图表达人、项目、技术、风险、文档之间的连接。
- 复利/信心评分页：用指标卡和证据链样式表达可信度。

### 4. 案例和实践页

案例页保留现有内容，但改成：

- 时间线：第 1-2 天、第 3-7 天、第 2 周后。
- 双栏对比：没有 Wiki / 有 Wiki。
- 操作步骤卡片：建立框架、定义规则、让 AI 帮忙、开始使用。

### 5. 图片和图示插入原则

- 封面和章节过渡页可以使用在线氛围图。
- 解释性页面优先使用内嵌 SVG 或 HTML/CSS 图示，保证信息可控且不依赖额外文件。
- 图片只服务于理解或氛围，不为了装饰而堆图。
- 避免新增本地图片资产，减少部署和维护成本。

## 非目标

- 不改动其他 Markdown 文档。
- 不新增 Slidev 主题或 npm 依赖。
- 不重写整套演讲内容。
- 不新增本地图片文件。
- 不改变项目部署方式。

## 验证方式

- `pnpm run build` 能正常构建。
- `slides.md` 保持 Slidev 可解析。
- 关键页在视觉上具备更清晰的层级和图示表达。
- 新增在线图片和内嵌图示不阻断本地构建。

## 实施注意事项

- 优先进行局部、可回退的 Markdown/HTML/CSS 修改。
- 保持中文表达自然，不为了视觉效果牺牲内容准确性。
- 如果某页文字密度过高，先通过布局分组解决；只有必要时才精简文字。
- 所有新增样式应限制在演示文稿内，避免影响仓库其他文档。
